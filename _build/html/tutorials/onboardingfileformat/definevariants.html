Définir des variantes dans un fichier d’intégration¶

  • Un produit peut avoir des variantes en termes de taille, de couleur et d’autres attributs. Une entreprise peut fabriquer ou vendre des articles qui sont essentiellement les mêmes, mais qui varient légèrement par leur couleur et leur taille.
  • Toutes les variantes d’un produit seront répertoriées dans une seule feuille d’un fichier Excel.
  • La feuille des variantes contiendra les détails suivants pour chaque variante par colonnes:
Détails Description
ID de variante Mentionnez l’identifiant unique de chaque variante dans cette colonne. Il n’y a aucune restriction sur l’en-tête de colonne/le nom de colonne.
ID d’article Mentionnez l’identifiant unique d’un article pour lequel vous définissez la variante. Elle aura la valeur de la colonne ID d’article de la feuille dans laquelle vous avez défini tous les articles.
Variante Le système comporte de nombreux types de variantes comme variante d’unité, variante de langue, variante embarquée, etc. Vous devez mentionner le nom de la variante comme en-tête de colonne. Par exemple, si vous souhaitez définir une variante d’unité, définissez l’en-tête de colonne comme Unité. Si vous définissez une variante d’unité, vous devez définir des sous-types de variante d’unité dans cette colonne. Pour ce faire, définissez des sous-types dans chaque ligne d’une feuille portant le même ID de produit. L’ID de produit restera identique dans chaque ligne lorsque vous définissez des sous-types de variante d’unité dans chaque ligne.
Attribut Vous devez mentionner le nom de l’attribut comme en-tête de colonne. Par exemple, si nous définissons la variante d’unité, ses attributs peuvent être EstUnitéBase, EstUnitéVente, etc. Vous devez indiquer la valeur de l’attribut à côté de chaque sous-type de variante d’unité dans chaque ligne.
