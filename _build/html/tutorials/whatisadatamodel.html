Qu’est-ce qu’un modèle de données?¶

Les produits du monde réel peuvent être décrits de diverses manières. Dans la vraie vie, nous pouvons dire qu’un téléviseur a des dimensions, des couleurs et un fournisseur. Cela regroupe toutes les données qui ont un rapport avec le produit. Le Modèle de données permet donc de structurer cette information afin qu’elle reflète fidèlement la réalité tout tentant de définir le rapport entre les différents éléments de Contentserv (produits, images, audio, couleur, etc.).

Exemple:

Un produit, un tee-shirt par exemple, peut être défini par un Modèle de données. Au plus haut niveau, on lui donne une classe Nature, « Produit » par exemple. Cette classe « Produit » permet de définir certaines variantes de ce T-shirt. À ce niveau, le T-shirt décrit la conception, il ne s’agit pas un objet vendable.

Le Modèle de données de votre T-shirt comprend également des classes Nature/attributs telles que « Vêtements ». La classe Attribut Vêtements est associée à différentes propriétés comme le tissu et la couleur au produit T-shirt. La Taxonomie est utilisée pour ajouter le produit T-shirt dans une classification de magasin plus large ou dans un système de catégories.
