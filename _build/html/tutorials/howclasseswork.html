Les classes¶

Qu’est-ce qu’une classe?¶

Dans Contentserv, il existe deux types de classes : (1) La classe « Nature » et (2) La classe « Non-Nature/atributs ». Les classes définissent les configurations de base de chaque produit. Chaque classe donnée à un produit permet de mieux le définir. Il existe plusieurs niveaux de classes et ceux-ci peuvent être utilisés conjointement pour obtenir une classification plus précise.

La classe la plus élevée est la classe « Nature ». Elle permet de définir le type de base de produit pour commencer. Dans Paramètres ou Aministration, vous pouvez personnaliser les classes Nature. Voici les différents types de classes proposées par défaut:


Les classes Nature par défaut¶

  1. Article individuel

    • Un produit à part entière vendable.

    • Exemple:

      Lit pour chien (palette de couleurs masculines/féminines).




  1. Produit

    • Un paramètre fictif pour un groupe de produits communs.

    • Exemple:

      1. Lit pour chien.



      1. Produit: Red Bull et ses variantes « Normal, sans sucre et Zero ».




  1. Pack personnalisable

    • Un groupe de produits dont la quantité peut être définie par le détaillant.

    • Exemple:

      Pack de bouteilles d’eau : 15 % de réduction si vous en achetez 10 ou plus. La réduction s’appliquera aux articles 11, 12, 13… etc.





  1. Pack fixe

    • Un groupe de produits dont la quantité de chaque produit est définie.

    • Exemple:

      Un kit de rasage qui comprend 3 lames et un tube de mousse à raser. Ces produits sont proposés dans un seul et même emballage, le client ne peut donc pas ajouter de produits.






  1. Ensemble de produits

    • Un ensemble de recommandations de produits comme des vêtements assortis.

    • Exemple:

      Chapeau, tee-shirt, shorts et sandales assorties.




  1. Offre promotionnelle

    • Un groupe de produits offerts à prix réduit.

    • Exemple:

      Achetez 3 boîtes de nourriture pour chiens pour le prix de 2.




Les classes d’attributs¶

(Aussi appelé Classes Non-Nature)

Les produits, définis uniquement par une classe Nature ne sont pas encore catégorisés dans votre PIM (gestion d’info. produit). Pour définir une catégorisation plus spécifique, vous pouvez ajouter des classes d’attributs. L’autre avantage apporté par l’utilisation d’une ou plusieurs classes d’attributs c’est qu’elles permettent de définir les propriétés d’un produit. Par exemple, un smartphone peut être associé à une « classe d’attributs mobiles », qui comprend différentes caractéristiques comme la taille de l’écran, la capacité de stockage, etc. Cette classe d’attributs pourra être associée à chaque produit de votre système. D’autres classes d’attributs peuvent être définies pour les produits de la section électronique ou informatique.

Voici quelques exemples de classes d’attributs et de caractéristiques potentielles qu’ils peuvent contenir. Chaque classe est entièrement personnalisable dans les Paramètres.

Exemples de caractéristiques de classe d’attributs¶

  1. Électronique
    • Tension



  2. Mobile (smartphone)
    • Taille d’écran


    • Stockage et RAM


    • Système d’exploitation


    • Caméra


  3. Aliments
    • Informations nutritionnelles



    • Contient de la noix?
    • Contient du porc?
    • Contient du bœuf?
