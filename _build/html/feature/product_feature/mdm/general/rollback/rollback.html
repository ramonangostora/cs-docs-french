Rollback¶

The Rollback feature allows a user to switch (or to revert) to a previous version of a product in the system.

Description:
  • The rollback allows a user to go back to any version if the user/supplier wishes to revert back to previous changes.
  • This feature can be accessed from within a product’s page under the ‘Timeline’ tab.
Related topics:
  • Manage Virtual Catalogs
  • Product Classifications
  • Compare Product Revisions
Application:

  • The Rollback feature is used to switch back to a desired version of a product.

  • This is an enhancement to the basic ‘Rollback’ feature, it allows the user add or remove any product relationship/context and return it back to the previous version.

  • Example:

    Supplier can add commercial text in the ‘description’ for Toblerone Chocolates during Holiday season. Once the season is over, he can rollback to the original description version.

How to do: Rollback¶

To do this, follow these steps:

  1. Go to PIM.


  1. Click to select the desired product for which you want to perform rollback operation (in this demo, we select ‘Orange bone toy’).


  1. Click on the Relationships tab.


  1. In this demo, we will replace the image and roollback to the previous image.






  1. Click on SAVE TO save to save the changes on the product.


  1. Go to Relationship tab.


  1. Go to the Timeline tab. The current version of the product would be with the latest version number on the top.




  1. Click on the version before we replaced the image (in this demo, we selected ‘version 0 27th March, 2018’).




  1. Once the rollback has been confirmed, Relationship Tab opens, and the relationship is restored.View the image (now rereverted to the old version).
