=============
Termes clés
=============

|

* Intégration
    L'intégration est le processus de transfert de données produits depuis le système d'un partenaire vers le PIM principal.
    Les informations du produit sont rassemblées dans un fichier puis téléchargées dans le système PIM du détaillant.
    Le PIM va récupérer les détails du fichier et les valider avec le MDM.

* Extraction
    Le processus d'extraction permet à l'utilisateur d'exporter des données produits à partir de Contentserv.
    Les données peuvent être téléchargées dans un fichier ou diffusées sur des canaux tels que le e-commerce.

* Utilisateur
    L'utilisateur est une personne qui se connecte au système Contentserv.

* Rôle
    Un rôle définit les autorisations d'un utilisateur dans le système. Une entreprise peut avoir plusieurs rôles tels qu'Administrateur ou Utilisateur général.
    Les rôles varient selon les entreprises, mais des rôles spécifiques à votre entreprise peuvent être créés.

* Flux de travail
    Un flux de travail est un processus commercial qui est modélisé numériquement dans Contentserv.
    Un exemple de flux de travail utilisé dans Contentserv est la gestion du cycle de vie des produits.

* Mappages
    Le mappage relie les différents éléments (*attributs, balises, classes et taxonomies*) définis dans un fichier d'intégration à leurs éléments respectifs (*attributs, balises, classes et taxonomies*) définis dans le système.

* Points de terminaison
    Le point de terminaison est une passerelle de données dans Contentserv.
    Des points de terminaison personnalisés peuvent être configurés pour l’importation et l’exportation de données.

* Point de terminaison entrant
    Le point de terminaison entrant est la passerelle par laquelle un fichier est importé dans le système.

* Point de terminaison sortant
    Le point de terminaison sortant est la passerelle par laquelle le fichier est exporté hors du système.

* Classe nature et non-nature
    *Classe nature:* Classe qui définit le type de base du produit. Ce type de produit est spécifique à la façon dont le produit doit se comporter dans le logiciel Contentserv.
    Les classes nature peuvent inclure *Article unique, Produit, Paquet ou d'autres types de produits personnalisés*.

    *Classe d'attribution:*  Classe qui ajoute des attributs spécifiques au produit auquel elle est affectée.
    Par exemple, une classe Téléphonie mobile peut ajouter des attributs pour *Taille de l'écran, Taille de la mémoire, Système d'exploitation,* etc.
