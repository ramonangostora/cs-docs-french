==============
Retour arrière
==============

La fonctionnalité de retour arrière permet à l'utilisateur de basculer vers (ou revenir à) une version antérieure d'un produit dans le système.

===================  =================================================================================================================================================================

**Description:**          * Le retour arrière permet à l'utilisateur de revenir à n'importe quelle version lorsque l'utilisateur/le fournisseur souhaite récupérer des modifications antérieures.
                          * Cette fonctionnalité est accessible dans la page d'un produit sous l'onglet « Chronologie ».

**Related topics:**       * :doc:`Manage Virtual Catalogs </feature/product_feature/mdm/products/catalogmanagementvirtualproductcatalogs/managevirtualcatalogs>`
                          * :doc:`Classifications de produits </feature/product_feature/mdm/governance/classificationandcategorization/improveproductclassifications>`
                          * :doc:`Comparer les révisions de produits </feature/product_feature/mdm/general/timelineandhistory/compareproductrevisions>`

**Application:**          * La fonctionnalité de Retour arrière est utilisée pour revenir à une version particulière d'un produit.
                          * Il s'agit d'une amélioration de la fonctionnalité de « Retour » basique, qui permet à l'utilisateur d'ajouter ou de retirer une relation/du contexte d'un produit et le faire revenir à la version précédente.
                          * *Exemple:*

                              Le fournisseur peut ajouter un texte commercial dans la « description » du Chocolat Toblerone pendant les fêtes de fin d'année. Après les fêtes, il peut revenir à la version originale de la description

===================  =================================================================================================================================================================

Marche à suivre : Retour arrière
================================
Pour ce faire, procédez comme suit:

#. Allez à **PIM**.

.. image:: /feature/_images/mdmimages/cs_rollbackb_clickpim.PNG
|
|

#. Cliquez pour sélectionner le produit pour lequel vous souhaitez exécuter un retour en arrière (dans cette démonstration, nous sélectionnons « Jouet orange en forme d'os »).

.. image:: /feature/_images/mdmimages/cs_rollbackb_selectproduct.PNG
|
|

#. Cliquez sur l'onglet *Relations*.

.. image:: /feature/_images/mdmimages/cs_rollbackb_clickrelationship.PNG
|
|

#. Dans cette démonstration, nous allons remplacer l'image et revenir à l'image précédente.

.. image:: /feature/_images/mdmimages/cs_rollbackb_replaceimage.PNG
|
|

.. image:: /feature/_images/mdmimages/cs_rollbackb_replaceimage2.PNG
|
|

.. image:: /feature/_images/mdmimages/cs_rollbackb_replaceimage3.PNG
|
|
#. Cliquez sur **ENREGISTRER SOUS** pour enregistrer les modifications apportées au produit.

.. image:: /feature/_images/mdmimages/cs_rollbackb_clicksave.PNG
|
|

#. Allez à l'onglet *Relation*.

.. image:: /feature/_images/mdmimages/cs_rollbackb_clickrelationship.PNG
|
|


#.  Allez à l'onglet *Chronologie*. La version actuelle du produit serait celle qui porte le numéro de la dernière version en haut.

  .. image:: /feature/_images/mdmimages/cs_rollbackb_clicktimeline.PNG
  |
  |

  .. image:: /feature/_images/mdmimages/cs_rollbackb_timeline.PNG
  |
  |

#.  Cliquez sur la version antérieure au remplacement de l'image (dans cette démonstration, nous avons sélectionné « version 0 27 mars 2018 »).

.. image:: /feature/_images/mdmimages/cs_rollbackb_selectversion.PNG
|
|

.. image:: /feature/_images/mdmimages/cs_rollbackb_clickrollback.PNG
|
|

#.  Une fois le retour en arrière confirmé, l'onglet Relation s'ouvre et la relation est restaurée. Vous pouvez voir l'image (revenue à l'ancienne version).

.. image:: /feature/_images/mdmimages/cs_rollbackb_viewimage.PNG
|
|
